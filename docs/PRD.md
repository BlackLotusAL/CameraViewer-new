# CameraViewer PRD

## 产品

- 使用 Python 3.9、PySide2、OpenCV 和 NumPy 实现桌面应用。
- 使用本地图像序列模拟三个独立相机，采用 View 驱动的异步逐帧 Pull。
- 数据源可替换为真实相机或 RPC，界面和调度接口保持不变。

## 数据

- 提供脚本生成 500 张 1920×1080 图像，文件名为 `frame_0001.jpg` 至 `frame_0500.jpg`。
- 支持命令行指定图像目录；采集前验证目录和完整序列。
- 三个相机共用图像目录，分别从第 1、168、335 帧开始循环读取。
- 每个相机维护独立游标；应用重启后恢复起始值。

## 采集

- 启动采集前不得读取或解码图像。
- 全局启动和停止统一控制三个相机。
- View 在相机进入运行状态时请求第一帧，在成功帧实际绘制后请求下一帧，在失败结果交付后继续请求。
- 窗口重绘不得重复触发帧请求。
- 成功和失败结果推进游标；作废结果不推进游标。
- 停止后保留最后一张成功图像；再次启动时从当前游标继续。

## 调度

- 每个相机最多有一个在途任务和一个合并重复需求的待提交任务。
- 使用最多 3 个线程的 `QThreadPool`；文件读取、解码和处理在 Worker 线程执行，GUI 更新在 GUI 线程执行。
- 三个相机独立调度，一路变慢或失败不得阻塞其他相机。
- 不得使用固定采集定时器、目标 FPS、帧队列、预取或最新帧缓存。
- 停止时清除待提交需求；迟到结果不得显示或推进游标。
- 停止后立即重启时，新需求必须保留至旧任务结算完成。

## 界面

- 同时显示三个等宽相机面板。
- 每个面板显示相机 ID、图像、状态、帧序号、分辨率和错误信息。
- 图像保持宽高比并居中，不拉伸、不裁剪；无图像时显示等待提示。
- 单路状态为未启动、采集中、已停止或错误；全局状态增加部分异常。

## 异常

- 目录或序列验证失败时，三个相机均进入错误状态。
- 单帧读取、解码或处理失败时，对应面板显示错误并继续采集。
- 连续 500 帧失败时，仅对应相机进入错误状态；其他相机继续运行。
- 一路错误时全局状态为部分异常，三路错误时为错误。
- 错误相机必须先停止再重新启动。

## 验收

- 连续运行 1 小时无崩溃、假死、未处理异常或线性内存增长。
- 连续启动和停止 50 次后无残留需求、任务或线程。
- 关闭窗口后 3 秒内退出，无残留进程或线程。
- 支持包含中文、空格和非 ASCII 字符的路径。
- 支持 Windows 10/11 和桌面 Linux。
- 自动化测试在严格警告模式下全部通过。
